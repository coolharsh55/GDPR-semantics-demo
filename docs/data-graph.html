Creating data graph from quantcast.com

consent collection dialogue

The consent collection dialogue on quantcast.com, as seen in Fig 1., starts with a "I Agree" and "Show Purposes". You can use the Web Archive version of this page here.

Clicking on "I Accept" assumes that the user has consented to all purposes and choices. This is evident by clicking on the "Change Consent" item at the bottom in the footer, and going into "Show Purposes" where all options have been pre-selected, as seen in Fig 2.

NOTE: We avoided the interpretation of whether the "I Accept" button fails some consent requirements such as not having options pre-ticked or pre-chosen by default. We focused on the modeling of options provided, with the ManualTest constraints responsible for this qualitative assessment.

Extracting Purposes, Processing, and Personal Data

Clicking on "Show Purposes" in the first screen (Fig 1.) takes the user to a second screen where various options are presented along with descriptions, as seen in Fig 3.

The second screen lists items such as "Information storage and access" along with a description next to a radio button that is off by default. We interpret the title as a purpose, and collect processing and personal data from the descriptions. Purposes are defined as both gdprov:Process and gc:Purpose, with processing defined as gdprov:Step and gc:Processing, and personal-data are defined using gdprov:PersonalData and gc:PersonalData. A list of purposes, processing, and personal data extracted from the consent dialogue are provided in Table 1.

For personal data mentioned in the consent dialogue, we indicated the source of this data as the user by defining a gdprov:DataCollectionStep that generates or collects this data.

Extracting Third-Parties

About halfway through the dialogue, a separate list of purposes associated with third-parties is provided, called "Third Party Vendors". Below the radio button, there is a "View Companies" link, which when clicked, expands the list of companies (third parties) associated with that purpose, as seen in Fig 4.

Clicking on the radio button is indicated to provide consent to all third parties as evidenced by the fact that individual third parties cannot be (de-)selected. Therefore, all third parties associated with a single purpose were represented using a single data sharing operation that shared data with ALL third parties listed. A list of purposes and third parties extracted from the consent dialogue are provided in Table 2. The third parties were defined using gdprov:ThirdParty and gc:ThirdParty. To associate them as data recipients for the specified third party, they were linked to a gdprov:DataSharingStep and gc:DataStep using the property gc:SharesDataWithThirdParty.

Representing Consent Choices

To represent the choices/options offered, the consent dialogue was defined using gc:Consent and gdprov:ConsentAgreementTemplate. Additionally, to signify multiple consent choices offered in the same dialogue (since the user could selectively give consent to each choice), the consent dialogue was defined as gdprov:ConsentAgreementTemplateBundle. Gathering more information from website

Additional information was collected by visiting the other pages on the website. These included information about the various products and services offered by Quantcast, as well as the privacy policy outlining data protection requirements of the organisation.

Information about products and services

Quantcast has five (broadly specifying) products/services listed on its website - Measure, Insights, Targeting, Measurement, and Quantcast Choice.

Measure is a free service that provides analytics regarding audience (visitors) to websites. It specifically mentions Demographics (age, gender, family, location, income, education, and occupation), Psychographics (purchase history, brand preference, cars driven, media consumption), Engagement (categorise visitors as passers-by, regulars, and fanatics), and Traffic (platform - web and mobile web, country, time period). Of these, we included data categories and types for Demographics and Psychographics in the data graph. We did not indicate their source as Quantcast does not provide any explicit information regarding the same on its website. For Psychographics, Quantcast specifies that it uses information from third-parties (Experian, Mastercard, DLX, TiVo, and Netwise) to 'augment' its profiles. We added the source for this data using this information.

The webpage provides a broad categorisation of such profiles in the form of Shopping Interests, Media Interests, Business & Occupation, Geography, and Political Interests. We added this as personal data by creating a subclass of gdprov:PersonalData for Demograhics and Psychographics, and defining each type as instances of this, including profiles.
